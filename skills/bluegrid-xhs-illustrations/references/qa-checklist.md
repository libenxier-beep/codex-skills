# QA Checklist

## 必过项

- 比例符合任务：小红书默认 4:5，文章正文默认 16:9。
- 背景是干净白底。
- 有小蓝格。
- 小蓝格是天蓝色圆角方块。
- 小蓝格戴细线蓝色耳机，耳机可见。
- 小蓝格承担核心动作，不只是装饰。
- 画面简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或动作流向。
- 红色只用于重点、问题、提醒或结果。
- 深蓝只用于耳机和少量补充说明。
- 画面有个人内容系统气质，不像通用素材图。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“Workflow / 系统架构图 / 常见坑 / 路线图”等标题。
- 耳机消失、太复杂、像主播麦克风、像头盔或像猫耳。
- 小蓝格像普通蓝色方块，缺少眼睛、肢体或动作。
- 小蓝格像儿童卡通、表情包或可爱吉祥物。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面没有“整理、收拢、判断、发布”这类动作感。

## 迭代方法

- 不够有 IP 感：强调天蓝圆角方块、黑点眼、细线四肢、深蓝细线耳机。
- 太可爱：强调 blank serious expression、not cute、not mascot、adult creator-tool sketch。
- 太普通：让小蓝格执行更具体的物理动作，如推抽屉、拉蓝线、贴标签、戴耳机降噪。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成一个低科技物件。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。
- 配件抢戏：让耳机变细、变小、变深蓝线稿，只保留识别功能。

## 交付判断

高质量图应该让读者在手机上先看到一个清爽的蓝色小角色，然后 1 秒内看懂这张图在讲什么。

如果第一眼像教程页、模板海报或通用蓝色插画，就不合格。
